## プロジェクト概要

「別画面を開く → コピー → 説明する → 依頼する」という既存AI利用の起動コストをゼロにする、
**ショートカット一発で画面文脈をAIが即座に把握し、短い指示だけで応答する相談UIアシスタント**。

自律実行エージェントではなく「提案のみ・読取専用」のパラダイムで、
企業セキュリティ・現場の不安・AIリテラシー不足という3つの壁を突破する。

## 機能要件

### FR-01: グローバルホットキー起動
- `Ctrl + Shift + Space` でアシスタントUIを即時起動する
- システム全体（他のアプリが前面でも）でキャプチャする
- 起動から「相談可能状態」まで **1秒以内**

### FR-02: 自動画面キャプチャ（事前コンテキスト取得）
- ホットキー押下と同時に現在の画面をキャプチャする
- **ユーザーが入力する前**にキャプチャ画像をGemini APIに送信し、AI側で状況を把握しておく
- AIが「準備完了」になった状態でユーザーとのセッションが始まる（毎回ゼロから説明不要）
- キャプチャ対象：現在アクティブなウィンドウ全体（または全画面）
- MVP段階では機密マスキングなし（Step 3で実装）

### FR-03: フローティングウィジェットUI
- 常に最前面に表示されるフローティングウィジェット
- ドラッグで任意の位置に移動可能
- 最小化ボタンあり（邪魔な場合に縮小可能）
- Otterキャラクター（アニメーション付き）を表示 — AI処理中は動くなど視覚的フィードバック

### FR-04: テキスト指示入力
- 短い指示を入力するテキストフィールド（例：「丁寧に」「短く」「何か変？」）
- Enter キーまたはボタンで送信
- MVP段階では音声入力なし（Step 2で追加）

### FR-05: AI応答表示 + コピー機能
- Gemini APIからの応答をウィジェット内に表示
- ワンクリックでクリップボードにコピーできるボタン
- 応答が長い場合はスクロール可能

### FR-06: セッション管理（MVP内）
- 同一起動セッション内での会話履歴を保持（短期記憶）
- アシスタントを閉じるとセッション終了・データ消去（デフォルト非保存）
- Step 2以降でセッション継続機能を追加

### FR-07: ローカルデータ保存
- 会話データはローカルのみ保存（クラウド送信なし）
- APIキーはローカル設定ファイルに保存（暗号化推奨）

### FR-08: 設定機能（MVP最小限）
- Gemini APIキー入力・保存
- ホットキーのカスタマイズ（将来拡張用に設計）

## 非機能要件

### NFR-01: パフォーマンス
- ホットキー押下 → ウィジェット表示：**500ms以内**
- 画面キャプチャ → Gemini APIへ送信完了：**1秒以内**
- AI応答表示（APIレイテンシを除く処理）：**500ms以内**
- アイドル時のCPU使用率：**1%以下**
- アイドル時のメモリ使用量：**100MB以下**

### NFR-02: 対応プラットフォーム
- **Windows 10 / Windows 11**（MVP）
- macOS対応はStep 2以降（必要に応じて）

### NFR-03: UX・アクセシビリティ
- AIリテラシーが高くない層でも直感的に使えるUI
- Otterアニメーションで処理状態を視覚的にフィードバック
- ウィジェットは作業の邪魔にならない最小フットプリント

### NFR-04: プライバシー・セキュリティ（MVP）
- 画面データ・会話データはローカルにのみ保持
- APIキーはローカル設定ファイルへ保存（プレーンテキスト保存を避ける）
- 機密情報マスキング：MVP段階では未実装（Step 3で対応）
- セキュリティ拡張ルール（SECURITY-01〜15）：**無効**（PoC・プロトタイプフェーズ）

### NFR-05: 保守性
- Pythonを主言語とし、開発者が自力でメンテナンス可能
- GUI部分は将来的なビジュアル強化（Otterアニメーション等）に対応できる設計

## 技術スタック（MVP推奨）

| 要素 | 推奨技術 | 理由 |
|---|---|---|
| 言語 | Python 3.11+ | 開発者が書けること、GCPライブラリが充実 |
| GUIフレームワーク | **CustomTkinter** または **PyQt6** | Otterアニメーションなどビジュアルに対応可能、Pythonネイティブ |
| 画面キャプチャ | **`mss`ライブラリ** | Windows対応、軽量・高速（全画面/ウィンドウ指定）、Pure Python |
| グローバルホットキー | **`pynput`** または **`keyboard`** | Windows対応、バックグラウンド常駐でキー検出 |
| LLM バックエンド | **Gemini API**（`google-generativeai`） | マルチモーダル対応（画像+テキスト）、Vertex AI移行も容易 |
| 設定保存 | `keyring` + JSON | APIキーをOSのキーチェーンに安全保存 |
| パッケージング | **PyInstaller** | Windows実行ファイル（.exe）生成 |

> **注記**: PyQtを選択した場合、QMLまたはアニメーションAPIで滑らかなOtterアニメーションが実現可能。
> CustomTkinterはよりシンプルだが、アニメーションはPillow + GIFで対応。

## AIバックエンド詳細（GCP/Gemini）

### 状況把握フロー（Reasoning モデルによる1コール完結）

```
[ホットキー押下]
      |
      v
スクリーンショット撮影
      |
      v
Gemini Reasoning モデルへ送信
      |
      v
[Thinkingプロセス（内部）]
  知覚：「これはGmailの返信画面、取引先へのメール」
  推論：「ユーザーはおそらく文面のトーン改善を求めるだろう」
  準備：文脈を内部状態として保持
      |
      v
「準備完了」状態でユーザーの入力待ち
      |
      v
ユーザー「丁寧に」と入力
      |
      v
文脈 + 指示 → AI応答生成
```

| 要素 | 内容 |
|---|---|
| **MVP使用モデル** | `gemini-2.0-flash-thinking-exp`（Reasoning対応・高速・低コスト） |
| **入力** | スクリーンショット画像 + テキスト指示 |
| **出力** | テキスト応答（修正文案・提案・要約等） |
| **Agent Engine** | 不要（単一 API コールで知覚→推論→応答が完結） |
| **将来移行先** | Vertex AI SDK（Step 3 クラウドバックエンド化時） |
| **APIキー管理** | ローカル設定ファイル（MVP）→ Vertex AI サービスアカウント（Step 3） |

## 成功基準（MVP）

1. `Ctrl+Shift+Space` を押すと1秒以内にウィジェットが表示される
2. AI（Gemini）がスクリーンショットを受け取り、画面内容を正しく把握できる
3. ユーザーが短い指示を入力すると、AI応答が表示される
4. 応答をコピーボタンでクリップボードにコピーできる
5. ウィジェットをドラッグで移動できる
6. アイドル時のCPU使用率が1%以下を維持する
